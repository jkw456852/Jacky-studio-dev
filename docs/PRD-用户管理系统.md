## 1. 产品概述

### 1.1 产品背景
为 Jacky-Studio / JK 平台构建一套完整的用户管理系统，实现用户的注册登录、信息管理、角色权限控制和密码重置功能。

### 1.2 目标用户
- 平台管理员：管理所有用户和权限
- 普通用户：使用平台各项功能

### 1.3 技术栈
- **前端：** React 19 + TypeScript + Zustand + React Router
- **验证：** Zod
- **样式：** Tailwind CSS

## 3. 用户流程图（文字描述）

### 3.1 注册流程
```
[访问注册页] → [填写信息] → [验证邮箱/手机] → [设置密码] → [注册成功] → [跳转登录]
```

### 3.2 登录流程
```
[访问登录页] → [输入账号密码] → [验证图形验证码] → [登录成功] → [跳转首页/工作台]
```

### 3.3 密码重置流程
```
[点击忘记密码] → [输入注册邮箱/手机] → [发送验证码] → [填写验证码] → [设置新密码] → [重置成功] → [跳转登录]
```

### 3.4 权限验证流程
```
[用户操作] → [前端路由守卫检查权限] → [有权限:执行操作] / [无权限:跳转无权限页]
            ↓
     [后端API] → [验证Token和权限] → [有权限:返回数据] / [无权限:返回401/403]
```

## 4. 数据模型

### 4.1 用户表 (User)
```typescript
interface User {
  id: string;
  email?: string;
  phone?: string;
  passwordHash: string;
  nickname: string;
  avatar?: string;
  bio?: string;
  roleId: string;
  status: 'active' | 'inactive' | 'deleted';
  createdAt: Date;
  updatedAt: Date;
}
```

### 4.2 角色表 (Role)
```typescript
interface Role {
  id: string;
  name: string;
  code: string; // 'admin' | 'user'
  description?: string;
  permissions: string[]; // ['user:read', 'user:write', ...]
  createdAt: Date;
}
```

### 4.3 验证码表 (VerificationCode)
```typescript
interface VerificationCode {
  id: string;
  userId?: string;
  type: 'register' | 'reset_password' | 'bind_email' | 'bind_phone';
  code: string;
  target: string; // 邮箱或手机号
  expiresAt: Date;
  usedAt?: Date;
}
```

## 5. 验收标准

### 5.1 用户注册/登录
- [ ] 用户可使用邮箱成功注册
- [ ] 注册时密码强度需满足：8位以上，包含大小写字母和数字
- [ ] 登录失败5次后需等待15分钟
- [ ] 图形验证码正确才能提交登录

### 5.2 用户信息管理
- [ ] 用户可查看和编辑自己的基本信息
- [ ] 修改邮箱/手机需发送验证码验证
- [ ] 注销账户需二次确认

### 5.3 角色权限
- [ ] 系统默认包含 admin 和 user 两种角色
- [ ] 管理员可为用户分配/变更角色
- [ ] 无权限用户访问管理页面时跳转到403页面
- [ ] 所有API接口都有权限验证

### 5.4 密码重置
- [ ] 验证码有效期10分钟
- [ ] 验证码发送间隔60秒
- [ ] 重置后原Token失效

## 6. 优先级排序

### 第一阶段 (MVP) - P0
1. 用户注册 + 登录
2. 基础角色权限（admin/user）
3. 密码重置（忘记密码）
4. 用户信息查看

### 第二阶段 - P1
1. 用户信息编辑
2. 权限验证（路由守卫）
3. 图形验证码
4. 登录限制

### 第三阶段 - P2
1. 角色管理界面
2. 账户注销
3. 多角色自定义

---

## 7. 风险与注意事项

1. **安全性**：所有密码必须加密存储，Token使用JWT
2. **验证码防刷**：限制发送频率和有效期
3. **数据隔离**：用户只能访问自己的数据
4. **响应式**：支持移动端访问
