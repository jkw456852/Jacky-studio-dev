最后更新：2026-04-24

适用范围：
- `pages/Workspace.tsx`
- `pages/Workspace/components/*`
- `pages/Workspace/controllers/*`
- `types/common.ts`
- 与树状节点渲染、生成、连线、折叠相关的工作区代码

执行约束：
- 严格遵循 [AI_DEVELOPMENT_STANDARD](../../standards/AI_DEVELOPMENT_STANDARD.md)
- 严格遵循 [工程协作规范](../../standards/工程协作规范.md)
- 本功能采用“先隔离、再静态 UI、再生成流、再连线”的主链路推进
- 禁止继续把原版节点与树状节点混合实现

## 1. 功能目标

在同一个画板中并存两套节点系统：

1. 原版节点系统：
- 保持现有经典节点交互与外观
- 原版图片节点、原版空生图节点、原版结果节点继续按经典方式工作

2. 树状节点系统：
- 新增树状图片节点
- 新增树状关键词节点
- 两者组成树状生成链路
- 使用与原版节点一致的底层能力与接口
- 但拥有完全独立的 UI、选中态、工具栏、连线与生成挂载逻辑

## 2. 核心原则

1. 同接口，不同节点壳：
- 树状节点与原版节点共用已有生图字段、参考图字段、模型字段、生成接口
- 绝不复用原版节点的主体 UI

2. 同画板共存，互不污染：
- 原版节点和树状节点可同时存在
- 任一系统的选中态、工具栏、连线、生成逻辑都不能污染另一套系统

3. 生成向下，分支向右：
- 关键词节点生成结果时，结果节点永远挂在下方
- 基于已有链路继续展开变体/分支时，分支永远向右

4. 单图与多图入口统一：
- 单图 `Tab`：在该图片节点下方创建关键词子节点并自动连线
- 多图 `Tab`：在下方创建一个聚合多张参考图的关键词子节点，不把多张图拆成多个父节点

## 4. 节点类型与数据定义

为避免继续混淆，树状节点需要显式数据标识。

建议新增字段：

- `nodeInteractionMode: "classic" | "branch"`：
  - 继续表示节点属于经典系统还是树状系统

- `treeNodeKind?: "image" | "prompt"`：
  - 仅树状节点使用
  - `image` 表示树状图片节点
  - `prompt` 表示树状关键词节点

- `treeNodeTone?: string`：
  - 树状关键词节点背景色标识

- `treeChildrenCollapsed?: boolean`：
  - 当前树状节点是否折叠其子树

说明：
- 原版节点不依赖上述树状专属字段
- 树状节点在渲染层必须优先根据 `nodeInteractionMode + treeNodeKind` 走独立分支

## 5. 节点行为定义

### 5.1 树状图片节点

- 表示一张图片资产
- 可作为树中的结果节点
- 可作为后续关键词节点的参考输入
- 选中时显示树状图片节点专属工具栏

### 5.2 树状关键词节点

- 表示一次生图意图
- 相当于树状模式中的 AI 生图节点
- 内部承载：
  - 参考图缩略图
  - prompt 文本
  - 作者与时间
  - 背景色
  - 折叠按钮

### 5.3 参考图输入

- 单图 `Tab`：
  - 在图片节点下方创建一个树状关键词子节点
  - 自动把该图片作为参考图写入关键词节点
  - 自动建立父子连线

- 多图 `Tab`：
  - 在下方创建一个聚合参考图的关键词子节点
  - 节点内部显示多张参考图缩略图
  - 不把多张参考图拆成多个父节点逐个连线

## 6. 生成流转规则

1. 树状关键词节点生成：
- 结果节点向下创建为树状图片子节点
- 自动连线

2. 树状图片节点继续派生：
- 单图 `Tab` 再次生成关键词子节点时，继续向下创建

3. 分支：
- 如果某节点已有主生成链，再新增平行派生时，分支向右创建

4. 原版节点：
- 保持原有生成与弹板逻辑
- 不接入树状节点生成挂载规则

## 7. 折叠与显示规则

1. 折叠入口：
- 仅树状关键词节点底部中间按钮可控制

2. 折叠行为：
- 隐藏当前节点下方整棵子树
- 父节点与当前节点保留显示
- 连线同步隐藏

3. 展开行为：
- 恢复该节点下游全部可见子节点与连线

## 8. 连线规则

1. 仅树状节点启用树状连线交互
2. 原版节点不出现树状连接口
3. 连线永远保持肉眼可见，不随缩放变得难以识别
4. 树状图片节点与树状关键词节点之间的连线样式统一
5. 参考 `video-workspace` 的连接口与拖线体验，但实现需限定在树状节点系统内部

## 9. 工具栏规则

### 9.1 树状图片节点工具栏

- 仅在树状图片节点被选中时出现
- 操作：
  - 编辑
  - 下载
  - 删除节点

### 9.2 树状关键词节点工具栏

- 仅在树状关键词节点被选中时出现
- 操作：
  - 背景色盘切换
  - 复制内容
  - 删除节点

说明：
- 原版工具栏与树状工具栏彻底分开
- 不允许树状节点继续走 `WorkspaceImageToolbar` 作为主体工具栏

## 10. 技术实施阶段

### 阶段 A：系统隔离

目标：
- 彻底拆开经典节点与树状节点的渲染分发

任务：
- 明确树状节点字段定义
- 在节点渲染层做独立 dispatch
- 让原版空生图节点彻底恢复原版链路
- 树状节点不再借用原版节点主体组件

验收：
- 原版节点恢复原版
- 树状节点逻辑不再影响原版节点选中态与工具栏

### 阶段 B：树状静态 UI

目标：
- 先把树状图片节点与树状关键词节点的外观做准

任务：
- 树状图片节点卡片
- 树状关键词节点卡片
- 紫色高亮边框
- 两套专属工具栏
- 色盘
- 折叠按钮

验收：
- 视觉与用户给定截图接近
- 非选中态、选中态、工具栏切换稳定

### 阶段 C：树状生成主链路

目标：
- 让单图、多图、生成结果方向全部正确

任务：
- 单图 `Tab` 下方创建关键词子节点并连线
- 多图 `Tab` 创建聚合关键词节点
- 关键词节点生成结果向下挂载树状图片节点

验收：
- 生成不再跑到右侧同级
- 多图参考不再拆成多个父节点

### 阶段 D：树状连线与折叠

目标：
- 完成树状模式专属连线交互

任务：
- 树状节点连接口
- 树状拖线
- 子树折叠/展开
- 连线与折叠联动

验收：
- 连线清晰可见
- 折叠后子树与连线一并隐藏

## 11. 开发顺序

固定顺序如下：

1. 先完成系统隔离
2. 再完成树状静态 UI
3. 再完成树状生成主链路
4. 最后完成树状连线与折叠

禁止跳过阶段直接拼接功能。

## 12. 验收清单

1. 原版空生图节点外观与交互仍为原版
2. 树状图片节点选中态符合图一
3. 树状关键词节点选中态符合图二
4. 两类树状节点非选中态符合图三
5. 树状节点选中时出现紫色高亮边框
6. 单图 `Tab` 直接在下方创建关键词子节点并连线
7. 多图 `Tab` 创建聚合参考图的关键词节点
8. 关键词节点生成结果向下挂载
9. 分支向右
10. 折叠按钮能控制子树显示
11. 原版节点和树状节点共存但互不污染
12. 树状图片节点工具栏符合图一
13. 树状关键词节点工具栏、色盘与折叠按钮符合图二

## 13. 本阶段执行策略

从本计划开始执行时：

1. 先落地树状字段与渲染隔离
2. 立即停止一切“共享一个节点壳再分情况打补丁”的写法
3. 所有树状 UI 走树状专属组件
4. 原版节点只允许走原版组件链路
