# 电商工作流图片文字覆盖系统开发排期版

## 1. 排期目标

本排期以“先解决乱码与返工，再逐步做效率和智能化”为原则，优先保证第一阶段能快速上线并真实改善步骤七体验。

建议采用 4 个阶段推进。

---

## 1.1 新增前置原则

后续电商一键工作流与上字系统必须统一遵守以下前置原则：

1. 步骤三补充提问只问会影响设计判断、证据表达和版式承载的关键信息。
2. 步骤五方案规划必须输出单图职责、信息任务、留白结构和组件预留，而不是只给题目和描述。
3. 步骤六提示词必须服务详情页底图生成，先保证主体识别、信息层级和可排版性，再谈风格。
4. 不再把固定模板或静默 fallback 当作主路径能力；主路径失败时应显式暴露问题。
5. 卖点必须尽量转成可见证据，整套图必须保持统一的色彩、材质、光影、镜头语言与说明图形语气。
6. 结果评审不能只看单张好不好看，必须同时判断商品一致性、单图职责完成度、证据是否成立，以及是否延续整套视觉系统。
7. 自动补全与上字编辑也必须遵循同一原则：补全失败不能静默装作成功；上字组件结构应优先跟随图的职责、留白与信息负载自动展开。
8. 上字系统不是事后贴字工具，而是最终落版系统；步骤六和步骤七必须优先生成可排版、可留白、可承载信息的无字底图。
9. 默认主路径应为“无字底图生成 -> 信息层级规划 -> 系统真实字体落版 -> 导出成片”，而不是“普通成图后再补一层字”。
10. 只有在创意海报、材质嵌字、透视融合字体等特殊场景下，才允许把“模型直接生文字”作为特例分支。

---

## 1.2 新增交付目标

本排期需额外覆盖一条关键目标：

要证明“无字底图 + 专业落版”不仅能解决乱码，还不能牺牲详情页一体性。

所以交付验收不能只看：

1. 能不能打字
2. 能不能导出
3. 有没有字体上传

还必须看：

1. 底图是否天然适合承载文字
2. 落版是否像详情页设计，而不是像后贴海报
3. 预览和导出是否保持一致
4. 标题、标签、参数、标注是否能形成统一层级
5. 是否真的比“AI 直接生字”更稳定、更可控、更省返工

## 2. 阶段划分总览

### Phase A：MVP 可上线版

目标：解决最核心的中文文字失控问题。

周期建议：5 到 7 个工作日

交付内容：

1. 字体上传
2. 字体注册
3. 步骤七单图上字入口
4. 单图文字编辑
5. 单图导出成片
6. 状态持久化恢复

### Phase B：效率增强版

目标：降低重复劳动，提高单人出图效率。

周期建议：4 到 6 个工作日

交付内容：

1. 自动换行
2. 字号自适应
3. 多图层编辑
4. 简单模板
5. 常用样式快捷应用

### Phase C：半智能排版版

目标：让系统能辅助排版，但不抢用户控制权。

周期建议：5 到 8 个工作日

交付内容：

1. 留白区域检测
2. 主体避让
3. 文字位置建议
4. 对比度建议
5. 可读性告警

### Phase D：批量生产版

目标：形成品牌化、模板化、批量化成片能力。

周期建议：6 到 10 个工作日

交付内容：

1. 多平台模板
2. 批量套版
3. 一套文案多图复用
4. 多版本导出
5. 品牌资产沉淀能力

## 3. 详细排期

## 3.0 先行改造：工作流目标重定义

预计：1 到 2 天

任务：

1. 明确步骤六的目标从“普通成图提示词”调整为“无字详情页底图提示词”。
2. 明确步骤五的输出必须包含留白规划、文字承载区、证据区、参数区、对比区等版式职责。
3. 明确步骤七的上字能力是最终落版系统，而不是独立小工具。
4. 补充评审维度：底图可落版性、信息承载性、成片一体性。
5. 为特殊场景单独保留“模型直接生字”的分支判断，但不作为默认主路径。

## 4. 依赖关系

### 4.1 必须先完成

1. `TextComposition` 数据结构
2. 字体注册服务
3. 步骤七结果卡的稳定结果 id
4. 本地项目状态持久化能力

### 4.2 可并行推进

1. 字体上传器与编辑器 UI 可并行
2. 渲染服务与 store 设计可并行
3. 模板数据结构与基础模板内容可并行

### 4.3 后置依赖

1. 智能排版依赖基础编辑器完成
2. 批量套版依赖模板系统稳定
3. 品牌级资产依赖项目级能力先跑通

---

## 5. 风险缓冲建议

1. Phase A 至少预留 1 天缓冲给字体与 canvas 兼容性。
2. 如果步骤七当前状态结构还不稳定，先固定 `resultId` 再接入编辑器。
3. 智能排版不要插队做，先把基础编辑链路打实。
4. 导出图源跨域问题要尽早验证，否则后面会卡上线。

## 7. 验收口径

### 技术验收

1. 中文字体渲染稳定
2. 预览导出一致
3. 刷新恢复正确
4. 不影响原有步骤七生图流程

### 产品验收

1. 文字问题不再依赖重生图解决
2. 单张图从改文案到导出显著更快
3. 版式统一性比纯 AI 生字更稳定
4. 默认结果看起来像详情页设计完成图，而不是“照片上补字”
5. 稀疏信息和丰富信息能自动使用不同复杂度的版式
6. 当底图不适合落版时，系统能明确暴露问题而不是悄悄硬贴
